Q: Is AzAd Smart Downloader free?

A: Yes! The extension is completely free and open-source. No subscriptions, no premium features, no ads. We believe in making useful tools accessible to everyone.

Q: Does the extension track me or collect my data?

A: No. We are 100% privacy-focused. All scanning happens locally in your browser. No data is sent to servers. No tracking, no analytics, no personal information collected. See our Privacy Policy for full details.

Q: How do I install the extension?

A: Visit the Chrome Web Store (link coming soon), click "Add to Chrome", and approve the permissions. The extension icon will appear in your toolbar. That's it!

Q: Why do I need to give the extension these permissions?

A:

  • activeTab: To scan the current webpage
  • scripting: To run the scanner on the page
  • downloads: To save files to your Downloads folder
  • storage: To save your preferences locally
We only use what's necessary and only when you click "Scan This Page".

Q: Can I download from any website?

A: The extension works on most websites, but some may block downloads or have security restrictions. If a file doesn't download, it may be restricted by the website itself, not our extension. Chrome's download rules will apply similar to manual downloads.

Q: How are files organized after download?

A: Downloads are organized in folders like: Downloads/website-name/images/file.jpg. This keeps your Downloads folder neat and organized by source and file type.

Q: Can I adjust download speed?

A: Yes! Go to extension options (right-click icon ‚Üí Options) to adjust:

  • Parallel connections (for speed vs. stability)
  • Retry attempts for failed downloads
  • Auto-scroll settings

Q: What file types does it support?

A: The extension scans for:

  • Images: JPG, PNG, GIF, WebP, SVG, BMP
  • Documents: PDF, DOC, DOCX, XLS, XLSX, PPT, PPTX
  • Videos: MP4, WebM, MOV, AVI, MKV
  • Audio: MP3, WAV, OGG, M4A, FLAC
  • Archives: ZIP, RAR, 7Z, TAR, GZ

Q: It's not finding files on a specific website. Why?

A: Some possibilities:

  • Files are loaded dynamically (JavaScript) and may need more time to load
  • Images are behind JavaScript obfuscation or CDN protection
  • The website actively blocks download tools
  • Files are fetched via API instead of direct URLs
Try clicking "Scan This Page" again or waiting a moment for dynamic content to load.

Q: Can I pause/resume downloads?

A: Downloads use Chrome's native download system. You can pause downloads in Chrome by clicking the download icon in your toolbar. Once closed, downloads can be resumed if the server supports partial requests.

Q: What happens to my scan results after I close the extension?

A: Scan results are temporarily stored in your browser's local storage and automatically cleared after download completes. Nothing is saved permanently unless you want to and the data stays only on your device.

Q: Is this extension open-source?

A: Yes! Check out our GitHub repository. You can review the code, contribute, or fork it for your own use.

Q: How can I report a bug or request a feature?

A: Open an issue on GitHub Issues. Please provide:

  • Steps to reproduce (for bugs)
  • Which website you're using it on
  • Your Chrome version
  • Any error messages

üîß Troubleshooting

Extension won't load or crashes

Try:

  • Reload the page (F5)
  • Disable and re-enable the extension (chrome://extensions)
  • Clear browser cache and cookies
  • Restart Chrome

Downloads not starting

Check:

  • Your Downloads folder is accessible and has write permissions
  • You're not running out of disk space
  • Chrome's download settings aren't blocking the downloads
  • Try adjusting download concurrency in extension options (lower = more stable)

Getting permission errors

The extension needs proper permissions. If you see errors:

  • Right-click the extension icon ‚Üí "Manage extension"
  • Ensure all permissions are granted
  • Reinstall the extension if needed

Files are corrupted or incomplete

This is usually a connection issue:

  • Check your internet connection
  • Try downloading individual files instead of bulk
  • Increase retry attempts in extension options
  • Lower parallel connections to increase stability
